ℹ️ About OFO

Online Food Ordering System

A Java-based OOP project demonstrating advanced software design concepts.

📚 OOP Concepts Implemented

1. Inheritance

User class → Customer and RestaurantOwner subclasses

PaymentMethod interface → CardPayment, WalletPayment, CODPayment implementations

2. Polymorphism

Different payment methods implement the same interface

Overridden methods like displayUserInfo() in subclasses

3. Encapsulation

Private members with public getters/setters

Data hiding and controlled access patterns

4. Abstraction

Abstract User class defining common user properties

PaymentMethod interface abstracting payment logic

5. Collections & Generics

HashMap for storing objects by ID

ArrayList for dynamic collections

Generic List interface for flexible data handling

📁 Project Structure

src/
├── models/
│   ├── User.java (Abstract Class)
│   ├── Customer.java (extends User)
│   ├── RestaurantOwner.java (extends User)
│   ├── Restaurant.java
│   ├── MenuItem.java
│   ├── Order.java
│   ├── OrderItem.java
│   ├── Review.java
│   ├── PaymentMethod.java (Interface)
│   ├── CardPayment.java (implements PaymentMethod)
│   ├── WalletPayment.java (implements PaymentMethod)
│   └── CODPayment.java (implements PaymentMethod)
├── handlers/ (Service Layer)
│   ├── UserService.java
│   ├── RestaurantService.java
│   ├── OrderService.java
│   ├── PaymentService.java
│   └── ReviewService.java
├── database/
│   └── Database.java (Singleton Pattern)
└── FoodOrderingServer.java (Main Server)
